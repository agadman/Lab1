Publicering

Vald publiceringsplattform

Jag har valt att publicera min webbplats via GitHub Pages. Det är en gratis plattform från GitHub som gör det enkelt att publicera statiska webbplatser direkt från ett Git-repo.

Kodstöd och språk

GitHub Pages stödjer HTML, CSS och JavaScript, vilket gör den perfekt för enklare frontend-projekt. Eftersom den inte kör någon backend-kod, passar den bäst för statiska webbplatser eller frontends som kommunicerar med API:er.

För- och nackdelar

  • Fördelar: Gratis, enkelt att använda, integrerat med Git-versionering, stabilt och säkert.
  • Nackdelar: Stödjer inte serverbaserad kod (t.ex. PHP, Node.js), viss konfiguration kan krävas för egna domäner.

Arbetssätt vid publicering

För att publicera webbplatsen laddar man upp sina filer till ett GitHub-repo, och aktiverar sedan GitHub Pages i projektets inställningar (Settings → Pages). När man pushar nya ändringar från Visual Studio Code till GitHub uppdateras webbplatsen automatiskt.

I mitt fall behöver jag först köra npm run build för att generera min färdiga output.css från Tailwind, innan jag pushar till GitHub. Bootstrap-sidorna använder CDN och kräver ingen extra byggprocess.
